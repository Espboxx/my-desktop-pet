<execution>
  <constraint>
    ## 桌面宠物项目的客观技术限制
    - **Electron架构约束**：主进程和渲染进程的严格分离，IPC通信的性能限制
    - **React Hook规则**：Hook只能在函数组件顶层调用，不能在循环或条件语句中使用
    - **TypeScript编译约束**：严格模式下的类型检查，any类型的使用限制
    - **性能基线要求**：60fps渲染性能，内存使用控制在合理范围内
    - **跨平台兼容性**：Windows、macOS、Linux平台的API差异和行为一致性
  </constraint>

  <rule>
    ## 强制性编码规则
    - **文件组织强制**：Hook必须放在对应的分类目录，组件必须有对应的样式文件
    - **命名约定强制**：组件PascalCase，Hook use前缀，常量UPPER_SNAKE_CASE
    - **导入顺序强制**：React → 第三方库 → 项目内部 → 类型定义，每组间空行分隔
    - **类型定义强制**：所有接口必须在petTypes.ts中定义，禁止内联类型定义
    - **状态管理强制**：全局状态必须通过PetStatusContext，禁止直接的useState全局状态
    - **错误处理强制**：所有异步操作必须有try-catch，关键功能必须有降级方案
  </rule>

  <guideline>
    ## 编码指导原则
    - **架构一致性优先**：新代码必须符合现有的架构模式和设计原则
    - **性能影响评估**：每次代码变更都要评估对渲染性能和内存使用的影响
    - **渐进式集成**：新功能通过现有的Hook和Context系统渐进式集成
    - **测试驱动开发**：关键功能必须有对应的测试用例和性能基准
    - **文档同步更新**：代码变更必须同步更新相关的注释和文档
    - **向后兼容保障**：新功能不能破坏现有功能的正常运行
  </guideline>
